Relatório do Trabalho 2 – INF1018 (Software Básico)

1. Proposta do Trabalho
O objetivo principal deste projeto foi desenvolver um microcompilador capaz de traduzir arquivos de texto contendo código na linguagem SBas para código de máquina executável na arquitetura x86-64.

Para alcançar este objetivo, o trabalho exigiu a aplicação prática dos seguintes conceitos:

- Geração de código em tempo de execução, onde o resultado da compilação é armazenado diretamente em um vetor de bytes.
- Invocação dinâmica do código gerado como um ponteiro de função, com suporte para receber até três parâmetros inteiros.

2. Implementação e Arquitetura do Sistema
A solução foi estruturada separando as responsabilidades de leitura e orquestração do processo de tradução de baixo nível.

A arquitetura final é composta pelos seguintes módulos principais:

Microcompilador (peqcomp.c): Atua como o núcleo do sistema. Lê o código fonte linha por linha, interpreta os comandos SBas e gera sequencialmente as instruções de máquina no vetor de código.

Orquestrador de Testes (testapeqcomp.c): É o responsável por orquestrar a execução, gerenciar a abertura dos arquivos de teste (como teste1.txt) e invocar o compilador.

A engenharia interna do compilador foi dividida da seguinte forma:

Manipulação da Pilha (Stack Frame): A geração de código inicia com o prólogo da função: salva-se o ponteiro de base antigo (push %rbp), move-se o topo da pilha para a base (mov %rsp, %rbp) e aloca-se espaço dinâmico (sub $32, %rsp), garantindo capacidade para até 8 variáveis locais de 4 bytes. Comandos de atribuição (vN : ...) inserem valores diretamente nessas variáveis na memória.

Mapeamento de Registradores e Aritmética: Operações matemáticas (vN = ... op ...) utilizam os registradores %eax e %ecx para a execução algébrica, guardando o resultado final de volta na memória. Para o comando de finalização (ret), o valor a ser retornado é movido para o registrador %eax precedendo o epílogo da função (leave seguido de ret).

Controle de Fluxo e Backpatching: Saltos condicionais (iflez) são convertidos para as instruções nativas cmp $0, %eax e jle. Como a posição exata da linha de destino pode ser desconhecida no momento da leitura, o compilador adota uma resolução postergada: registra as informações dos saltos pendentes e, ao final da leitura, calcula e injeta os offsets definitivos cruzando os dados com o mapeamento do início de cada linha.

3. Validação e Análise dos Testes
O funcionamento do compilador foi atestado utilizando a bateria de testes padrão e um cenário extremo adicional:

Teste 1: Retorno de constante simples (ret $100), compilado e validado com sucesso.

Teste 2: Atribuição via parâmetro p1 seguida de incremento. O comportamento esperado de retorno do novo valor foi confirmado.

Teste 3: Validação do desvio condicional (iflez) e incremento, computando a ramificação lógica esperada (retornando 0 ou 1) de maneira íntegra.

Teste 4: Uso concorrente de múltiplas variáveis e operações encadeadas, gerando o cômputo correto da expressão.

Teste 5: Emulação de iteração de laço (similar a um cálculo de fatorial). As diretivas de múltiplos saltos condicionais (iflez) reproduziram o resultado matematicamente esperado.

Teste 6 (Caso Limite): Ao definir v0 como 0 e tentar saltar para a linha 4 precocemente, o sistema reportou uma falha de segmentação (segmentation fault). A análise diagnosticou que saltar para linhas ainda não registradas no vetor de posições resulta no cômputo de offsets incorretos, enviando a execução para áreas inválidas da memória. Adicionalmente, identificou-se que o compilador não previne a execução de comandos subsequentes a uma instrução ret.

4. Instruções de Compilação e Uso
O projeto foi codificado com apoio do Visual Studio 2022, mas estruturado para compilação estrita em ambiente Unix/Linux via WSL.

Para compilar, abra o terminal na raiz dos arquivos e execute:
gcc -Wall -Wa,--execstack -o testapeqcomp testapeqcomp.c peqcomp.c

Para rodar os diferentes cenários:

Certifique-se de que os arquivos .txt de teste estão no mesmo diretório do orquestrador.

Altere o nome do arquivo na chamada fopen dentro de testapeqcomp.c para o arquivo alvo.

Recompile utilizando o comando acima e execute o binário resultante.

5. Conclusão
Os testes comprovaram que a estrutura geral do compilador é altamente funcional. O sistema traduz de forma exata a lógica nativa da linguagem SBas para código x86-64, dominando o uso do Stack Frame, registradores e operações básicas de fluxo. Contudo, a incidência de falha no caso extremo demonstra a necessidade de aprimorar a robustez do software, recomendando-se a implementação de validações extras de integridade (como impedir o desvio para linhas inválidas e alertar sobre código inacessível) para garantir uma execução tolerante a falhas.
